## 1. 风格定义

使用“macOS 原生感 + 专业数据工作台 + 暖中性极简主义”。界面应安静、可信、紧凑、可审计，像研究运营控制台，不像营销落地页或高饱和科技大屏。

关键特征：

- 暖灰画布和低饱和表面。
- 黑白高对比主操作，功能色只表达状态。
- 细边框、极弱阴影、克制圆角。
- 系统字体承载说明，等宽字体承载事实与技术元数据。
- 桌面端高信息密度，窄窗按业务顺序堆叠。
- 状态、证据、成本、来源、限制和下一步操作优先于装饰。

## 2. 品牌与色彩

### 品牌

- 正式英文名只能写作 `Sortlytic`。
- 应用、README、官网和发布物复用 `apps/macos/src-tauri/icons/icon.png`。
- Logo 的青色与黄色用于品牌识别，不把它们滥用为所有组件的背景。
- 使用真实 macOS 红黄绿窗口控制点；不得用 Emoji 或近似图形替代。

### 语义色

- `success`：成功、已连接、校验通过、可用。
- `warning`：需确认、额度或成本提醒、部分成功。
- `danger`：失败、阻塞、删除和不可逆确认。
- `info`：中性进行中状态和一般信息。
- `focus`：键盘焦点，不作为普通装饰色。
- 同一状态同时提供文字或图标，不只依赖颜色。

## 3. 字体与数据排版

- 正文字体：`-apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif`。
- 页面标题：28px、650、行高 1.15、字距 `-0.025em`；窄窗降为 24px。
- 区块标题：15px、650、行高 1.4。
- 卡片标题：13px、650、行高 1.55。
- 正文和说明：11–12px；不得为了“高级感”把大量正文压到不可读尺寸。
- 上下文标签：10px、650、等宽字体。
- ID、路径、金额、时间、版本、代码和事实数字使用等宽字体与 `tabular-nums`。
- 信息层级遵循“页面标题 > 区块标题 > 关键事实 > 正文 > 辅助元数据”。

## 4. 几何、表面与间距

- `--radius-sm: 4px`：选项、紧凑内部元素。
- `--radius-md: 7px`：按钮、输入框、状态带、次级容器。
- `--radius-lg: 12px`：主面板、任务卡、弹窗。
- `--radius-pill: 999px`：短状态标签，不用于大面积按钮。
- 应用前端容器圆角为 16px；原生窗口圆角需要独立实机验收。
- 主按钮、次按钮和工具按钮最小高度 40px；应用内选择器通常为 42px。
- 常规区块间距使用 16px；主面板内边距通常为 20px，窄窗降为 14px。
- 浅色面板阴影只用 `0 1px 2px rgb(31 28 22 / 0.04)`；深色主题默认无面板阴影。
- 依赖背景明度、1px 边框和留白建立层级，不重复嵌套同色灰卡。

## 5. 页面布局

- 桌面应用外壳：224px 侧边栏 + 流式工作区。
- 工作区最大宽度 1720px，水平内边距在 20–32px 之间。
- 顶栏包含页面上下文、主标题、说明、当前状态和工具按钮。
- 顶栏上下文必须随“首页、新建任务、任务、设置、使用指南”变化。
- 没有检查栏的页面使用单栏；只有确有证据、详情或检查内容时使用 `主内容 + 320–356px 检查栏`。
- 页面主标题表达当前页面，工作区名称只能作为次要上下文。
- 首页只显示真实工作区数据或明确空状态，不回退到演示数据。
- 新建任务、任务列表、任务结果和设置应保持独立信息架构。

## 6. 组件规则

### 面板与事实网格

- 主面板使用实体 `surface`、细边框、12px 圆角；历史类名 `glass-panel` 不代表玻璃拟态。
- 同一事实网格的所有相邻项目使用连续且可见的统一分隔。
- 浅色主题中的主指标不得单独反相为纯黑卡片，除非具有明确选中或告警语义。

### 按钮

- 主按钮：`primary` 背景和 `primary-ink` 文字。
- 次按钮：`surface` 背景、`border` 边框。
- 工具按钮：40×40px，配 15–18px Lucide 图标。
- 按下反馈可缩放到 0.96；禁用状态取消按下变换。
- 危险按钮只在删除或明确不可逆确认时使用 `danger`。

### 状态标签

- 使用 22px 左右的紧凑胶囊标签。
- 标签由语义色、浅色语义表面、图标和文字共同表达。
- 不用状态标签替代长篇错误解释；阻塞原因需显示可执行的修正建议。

### 表单和选择器

- 字段标签约 11px、650；输入控件最小高度 40px。
- 错误贴近对应字段，并绑定 `aria-describedby`、`aria-invalid`。
- 平台、国家地区和主流程选择必须使用 `AppSelect` 一类应用内列表框。
- 长列表提供搜索；焦点、当前项和选中项必须可见，窄窗弹层不得溢出。
- 新建采集任务不预填具体任务实例；示例只能作为不参与提交的占位提示。

### 任务和结果

- 默认任务卡保持干净；用户进入批量管理模式后才显示选择框、全选和批量删除。
- 选中态反馈作用于整卡，不只改变选择框。
- 取消只停止执行并保留记录；删除二次确认并清理关联本地数据。
- 编辑、取消、运行、查看结果和导出放在清晰的横向操作区，不挤入固定窄列。
- 成功和部分成功任务提供应用内结果入口，并区分加载、空结果、读取失败和真实结果。
- 导出路径与打开动作组成紧凑状态带，不使用调试式裸路径输入框。

### 弹窗与系统反馈

- 弹窗使用明确标题、说明、关闭按钮、焦点管理和 `aria-modal`。
- 遮罩保持克制，弹窗使用 12px 圆角和清晰层级，不做全屏炫技转场。
- 加载、成功、错误和实时状态分别使用 `status`、`alert`、`aria-live` 等合适语义。

## 7. 内容与业务状态

- 内容顺序优先采用“状态 → 事实 → 操作 → 证据 → 技术元数据”。
- 真实采集、预算、余额、证据、Schema、提示词版本、日志和导出结果必须可追溯。
- 缺失值区分“任务未设置”和“未采集到”；数值 `0` 必须保留为 `0`。
- 未读取到金额时显示不可用，不得伪装为零或沿用历史成功值。
- 成功状态必须来自经过解析并落库的真实响应，不能由按钮点击或兼容空数组推断。
- UI 文案进入现有 i18n 资源，不在组件中散落新的硬编码业务文案。

## 8. 响应式

- `1320px`：侧边栏收窄至 208px，检查栏约 320px。
- `1180px`：侧边栏约 196px，双栏页面改为单栏。
- `960px`：左侧导航改为顶部横向导航。
- `760px`：顶栏、操作区和双列表单换行。
- `560px`：导航隐藏文字、工作区内边距降至 12px、面板内边距降至 14px。
- 最小宽度为 320px；所有控件保持可达且不横向溢出。
- 表格可以在自身容器内横向滚动，但页面主要操作不得依赖不可见的水平滚动。

## 9. 可访问性与动效

- 提供跳过导航链接、语义标题、区域标签和键盘操作路径。
- 所有交互元素使用清晰的蓝色 `focus-visible` 描边与偏移。
- 自定义选择器实现 `combobox/listbox/option` 语义和方向键、回车、Esc 操作。
- 进度条暴露数值；异步结果使用 `aria-live`；错误使用 `alert`。
- 常规过渡为 120–140ms，使用快速进入、柔和停止的缓动。
- 支持 `prefers-reduced-motion`，禁用非必要动画和长过渡。
- 隐藏滚动条时仍需通过布局、裁切和内容结构让可滚动性可理解。

## 10. 禁用模式

- 禁止把 UI 改成霓虹、赛博朋克、高饱和科技大屏或营销落地页风格。
- 禁止大面积渐变、毛玻璃、发光描边、强投影和无业务语义的彩色卡片。
- 禁止为了“现代感”随意增大圆角、留白和字号，导致桌面信息密度下降。
- 禁止用纯黑反相卡片突出普通浅色事实指标。
- 禁止重复嵌套同色灰卡、裸浏览器表单控件和调试式路径框。
- 禁止使用 Emoji 作为图标，禁止伪造 Logo，禁止让演示数据进入正式首页。
- 禁止只验证浏览器开发服务器后宣称 macOS 桌面 UI 完成。

## 11. 验收清单

### 视觉

- [ ] 暖中性表面、细边框、克制阴影和圆角保持一致。
- [ ] 品牌 Logo 来自实际打包资源，语义色没有被装饰性滥用。
- [ ] 页面标题、上下文、事实、状态和操作层级清晰。
- [ ] 浅色和深色主题都没有突兀的反相块、边界丢失或对比不足。

### 交互

- [ ] 默认、悬停、按下、焦点、禁用、加载、空、成功、警告、失败和危险确认均有设计。
- [ ] 键盘可以完成导航、列表框选择、弹窗关闭和主要操作。
- [ ] 危险操作有清晰二次确认，取消与删除语义不混淆。

### 响应式

- [ ] 至少检查桌面宽度和 560px 以下窄窗。
- [ ] 表单、任务操作栏、结果视图和弹窗在窄窗不溢出。
- [ ] 导航切换为顶部形态后仍能访问全部页面。

### 桌面实机

- [ ] 结束旧实例，启动本次打包产物并核对 PID 对应的绝对路径。
- [ ] 截图包含窗口外侧桌面背景。
- [ ] 四角放大后直接透出桌面，无黑色、白色或其他不透明矩形底层。
- [ ] 本次产物的窗口圆角、主题、滚动和原生控制点均符合预期。
