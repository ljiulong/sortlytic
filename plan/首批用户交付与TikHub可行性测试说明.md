# 首批用户交付与 TikHub 可行性测试说明

日期：2026-07-17
适用对象：第一批非开发测试用户、产品验收、外部协作方
适用版本：macOS 单端 MVP，本地工作区模式

本文是独立的交付与实机验收指南，不并入产品路线图或实现计划。

## 1. 交付产物

本次可交付产物位于：

- 应用包：`apps/macos/src-tauri/target/release/bundle/macos/Sortlytic.app`
- 安装包：`apps/macos/src-tauri/target/release/bundle/dmg/Sortlytic_0.1.0_aarch64.dmg`

建议给非开发用户分发 `.dmg`。测试机打开后，如 macOS 提示来自未认证开发者，可在“系统设置 > 隐私与安全性”中允许打开。这个首次打开提示与旧版钥匙串授权窗不是同一类提示。

## 2. 首次使用与 API 配置

1. 打开 `Sortlytic.app`，应用会自动创建或打开默认本地工作区。
2. 进入设置页。API 区域只提供“配置 TikHub API”和“配置 AI API”两个按钮，配置详情不在设置页内联展示。
3. 点击任一按钮后，弹窗会先显示该类型已保存的配置列表。列表包含配置名称、端点或模型、脱敏密钥、校验状态和当前配置标记。
4. 从旧版本升级时，已有 TikHub 与 AI 元数据会显示为“需重新输入”。应用不会读取或删除旧 macOS Keychain 条目，用户需要在对应配置中重新输入一次密钥。
5. 配置 TikHub 时，填写唯一名称并选择 API 域名：
   - 国际网络优先选择 `https://api.tikhub.io`。
   - 中国大陆网络不稳定时选择 `https://api.tikhub.dev`。
6. 粘贴 TikHub Token，保存后执行真实账户连通测试。测试成功后应显示脱敏账号、余额、免费额度、总可用额度和今日用量等安全摘要。
7. 配置 AI 时，选择供应商和 API 格式，填写 Base URL、默认模型与 API Key。Ollama 可以不填写密钥。AI 本次只检查配置、协议、地址、模型和密钥是否完整，不发起真实模型请求。
8. 每类第一条验证成功的配置会自动设为当前配置。以后新增的配置只保存，不会替换当前配置；需要在列表中点击“设为当前”完成切换。
9. 编辑配置时，密钥留空表示保留原密钥。界面始终只显示首尾脱敏值，不提供查看或复制完整密钥的入口。

TikHub 与 AI 都可以保存多条命名配置。可保存多个 TikHub 账号，也可保存同一 AI 供应商的多个账号或端点；每类最多一条当前配置。

此前使用 TikHub 已登录账号和已有 API Key 完成了可行性测试，未创建新的持久 API Key。

- API 密钥页：`https://user.tikhub.io/dashboard/api`
- 测试时 API Key 状态：活跃
- 测试时免费额度：`$0.0500`
- 测试时余额：`$0.0000`
- 测试时邮箱状态：已验证

应用内连通性测试调用：

- `GET /api/v1/tikhub/user/get_user_info`
- `GET /api/v1/tikhub/user/get_user_daily_usage`

免费 demo 可行性测试调用：

- `GET /api/v1/demo/tiktok/web/fetch_user_profile`
- `GET /api/v1/demo/douyin/web/fetch_one_video`

测试结果：

- TikHub 官方 API 域名 `https://api.tikhub.io` 可连通。
- TikHub 用户信息接口返回成功，测试期间免费额度保持 `0.05`。
- TikTok demo 与抖音 demo 端点均返回 HTTP 200。
- 官方 OpenAPI 描述中 demo 端点标注为免费使用，本次 demo 调用后免费额度仍为 `0.05`。

这些结果用于证明接口与免费 demo 可用，不代表新版本已经完成本指南第 8 节的打包实机验收。交付前仍需用本次打包产物和当前工作区重新验证。

## 4. 配置存储与任务绑定行为

API 配置统一保存到当前工作区：

`~/Library/Application Support/com.steven.sortlytic/default-workspace/secrets/api-config.json`

这个 JSON 是 TikHub 与 AI 配置的唯一主数据源。SQLite 只保留可重建的运行镜像、密钥引用元数据、审计记录和不可变任务快照，不保存明文密钥。

任务使用 TikHub 配置时遵循以下规则：

- 新任务入队时不冻结 TikHub 配置。执行器真正领取任务时，才绑定当时的当前 TikHub 配置。
- 已排队且从未运行的任务会使用领取时的当前配置，因此切换配置后不需要重新创建任务。
- 已经运行过并进入恢复队列的任务继续使用原运行快照，不跟随当前配置切换。
- 运行快照保存配置身份、修订号和密钥引用，不保存明文密钥。运行时通过快照引用从 JSON 读取对应凭据。
- 被运行中或恢复中快照引用的 TikHub 配置不能编辑或删除，但可以切换到另一条当前配置。
- 删除当前配置后，当前指针保持为空，不会自动选择其他配置。用户明确选择新的当前 TikHub 配置前，新任务不会被领取执行。
- JSON 缺失、损坏、权限不合格或没有当前 TikHub 配置时，API 管理和新任务领取会失败关闭，历史数据与本地工作区仍可浏览。

当前任务队列只使用 TikHub 配置。AI 当前配置只用于保存、校验、切换和未来模型执行；自然语言计划仍由本地规则引擎生成，不调用 AI API。

## 6. 安全与隐私说明

- 密钥以明文写入当前工作区私有 JSON，不进入 SQLite、日志、导出或 Webhook。
- `secrets` 目录权限固定为 `0700`，`api-config.json` 文件权限固定为 `0600`。应用拒绝符号链接、非普通文件、宽松权限以及无法落实私有权限的文件系统。
- JSON 采用同目录临时文件、文件同步和原子重命名写入。损坏或无法解析的原文件不会被新内容覆盖。
- UI 只展示脱敏 Token，例如 `hcoO...[REDACTED]...YA==`。完整密钥不进入前端缓存，也不提供查看或复制入口。
- `0700` 和 `0600` 可以阻止其他本机用户读取，但不能防止同一 macOS 账户下的恶意进程访问明文密钥。这是停用 Keychain 后必须明确接受的安全降级。
- 应用迁移旧配置时只导入 SQLite 中的非敏感元数据，不读取或删除旧 Keychain 条目，因此迁移过程不会触发钥匙串授权窗。
- 旧配置会标记为“需重新输入”，用户重新填写 TikHub Token 或 AI API Key 后才能测试和激活。确认新版本正常后，可由用户自行在“钥匙串访问”中删除旧条目。
- 配置 JSON 不进入应用备份、Excel、PDF、日志、Webhook 或前端文件访问范围。
- 测试后应清空系统剪贴板，避免粘贴过的密钥残留。
- 免费 demo 调用只用于验证连接和返回结构，不代表已完成真实业务采集。

## 7. 已知限制

- 当前 MVP 仅支持 macOS 单端本地工作区。
- 当前未实现用户账号、远端数据库、远端同步或多设备自动同步。
- JSON 内保存明文密钥，权限隔离能力不等同于 macOS Keychain。
- JSON 是应用内部存储接口，不承诺支持用户手工编辑；手工修改可能导致配置管理和新任务领取失败关闭。
- AI 配置目前不执行真实模型连通测试，也不参与自然语言计划生成。
- 真实采集调用仍需在成本上限、端点价格和任务参数确认后执行。
- 小红书暂无本次使用的官方 demo 端点；本次只完成 TikTok 与抖音 demo 验证，以及 TikHub 用户与额度接口验证。

## 8. 打包应用实机验收

交付给非开发用户前，必须按以下顺序使用本次打包产物验收：

1. 结束所有开发版应用、旧版 Sortlytic 和使用相同 bundle identifier 的进程。
2. 启动本次 `target/release` 生成的打包产物，查询进程 PID，并核对 PID 对应的可执行文件绝对路径确实位于本次产物内。
3. 打开包含旧 SQLite 配置和 Keychain 引用的现有工作区，不要改用空白测试工作区规避迁移场景。
4. 进入设置页，确认 API 区域只有“配置 TikHub API”和“配置 AI API”两个按钮。打开两个管理弹窗，确认都先显示配置列表，旧配置显示“需重新输入”。
5. 重新输入现有 TikHub Token，保存并完成真实连通测试；重新输入自定义 AI API Key，保存并完成本地完整性校验。全过程不得出现钥匙串或 API 密钥访问权限窗。
6. 检查 `secrets` 目录与 `api-config.json` 已生成，目录权限为 `0700`，文件权限为 `0600`。确认明文测试哨兵只存在于 JSON，不出现在 SQLite、日志、审计、导出或界面中。
7. 显式选择当前 TikHub 配置，创建任务并运行完成。随后再次运行任务，确认两次运行都不出现钥匙串授权窗。
8. 分别新增第二条 TikHub 配置和第二条 AI 配置，确认新增配置不会自动替换当前配置。手动切换后，每类都只有一条当前配置标记。
9. 切换当前 TikHub 配置后，确认尚未领取的新鲜排队任务绑定新配置；已经运行过并进入恢复队列的任务继续使用原快照。
10. 完全退出并重新打开应用，确认两类配置列表、当前标记、脱敏密钥、端点、模型与校验状态保持不变。
11. 确认 AI 测试没有发起真实模型请求，生成自然语言计划时仍使用本地规则引擎。
12. 确认 Excel 与 PDF 导出可在本机打开，且不包含完整 API Key、Authorization Header 或其他敏感 Header。

任何一步出现旧进程误启动、PID 路径不匹配、配置权限不合格、完整密钥泄漏、当前配置状态异常或钥匙串授权窗，都不能视为验收通过。
