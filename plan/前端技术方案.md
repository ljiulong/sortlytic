# 前端技术方案

版本：v0.2  
日期：2026-07-05  
关联 PRD：`智能数据整理平台_PRD.md` v0.6  
文档定位：定义 macOS MVP UI、交互、前端状态、组件体系、视觉规范、前后端通信和前端验收标准，并保留后续多端扩展原则

## 1. 前端目标

前端要让用户完成从配置、采集、整理、预览到导出的完整工作流，同时避免 AI slop 和复杂数据界面的认知负担。

MVP 前端目标：

- 支持 macOS 桌面端完整工作台。
- Windows、iOS、Android 不进入 MVP，后续作为独立本地工作区扩展。
- 支持 TikTok、抖音、小红书三平台采集入口。
- 支持表单式采集和自然语言智能采集双入口。
- 支持模型供应商配置和自定义 OpenAI-compatible endpoint。
- 支持 MVP 提示词优化、提示词版本管理和基础回归样例入口。
- 支持原始数据、标准化数据、AI 结果、来源追溯并排或联动查看。
- 支持 Excel 和 PDF 导出预览、状态和错误处理。

## 2. 技术选型

推荐：

- React
- TypeScript
- Vite
- Tailwind CSS
- Radix UI 或同类无样式基础组件
- TanStack Query
- Zustand
- TanStack Table
- ECharts
- Zod 或同类 Schema 校验库
- React Hook Form 或同类表单库

选择理由：

- React 适合组件化数据工作台。
- TypeScript 提高模型配置、任务状态、Schema 和导出模型的类型安全。
- Vite 保持开发体验轻。
- Tailwind CSS 适合建立统一设计系统。
- TanStack Query 适合管理 Tauri commands 返回的异步状态。
- Zustand 适合管理轻量 UI 状态。
- TanStack Table 适合桌面端复杂表格。
- ECharts 适合趋势、情绪和平台对比图表。

## 4. 视觉设计指南

用户要求前端使用纯色科技风、玻璃拟态、圆润圆角卡片。

### 4.1 总体风格

- 纯色科技风：以清晰、冷静、专业为主，不做复杂插画和营销式英雄页。
- 玻璃拟态：用于侧栏、顶部栏、浮层、详情抽屉和少量关键面板，避免全页面过度模糊。
- 圆润圆角卡片：卡片圆角建议 8px 到 12px，工具按钮可 8px。
- 信息密度：macOS 端偏工作台，保持高信息密度；后续移动端再用卡片和折叠降低拥挤。
- 不使用单一紫蓝渐变统治界面，避免一眼像通用 AI 模板。

### 4.2 色彩建议

基础色：

- 背景：深色石墨黑、浅色冷白。
- 主色：电光蓝或青蓝，用于主要操作和状态聚焦。
- 辅助色：青绿、琥珀、玫红，用于成功、警告、风险和分类。
- 中性色：灰阶用于表格、边框、分割线和说明文字。

状态色：

- 成功：绿色。
- 运行中：蓝色。
- 等待确认：琥珀色。
- 部分成功：青色加琥珀提示。
- 失败：红色。
- 待人工确认：紫色或琥珀色，但要控制使用面积。

### 4.3 组件风格

- 卡片只用于任务、记录、报告、设置组和浮层，不要卡片套卡片。
- 表格容器要稳定尺寸，不因 hover 或动态标签产生布局跳动。
- 玻璃模糊必须有纯色兜底，关闭模糊后仍可读。
- 所有按钮文字必须在 macOS 窄窗口下不溢出。
- 工具按钮优先图标加 Tooltip，常用图标使用 lucide 或现有图标库。

## 5. 响应式策略

### 5.1 macOS 桌面端

macOS 端布局：

```text
左侧导航
  |
中间主内容区
  |
右侧详情 / AI / 日志抽屉
```

适用：

- 多列表格。
- 并排查看原始数据和 AI 结果。
- 来源追溯。
- 长任务运行日志。
- 报告预览。

### 5.2 后续移动端

移动端不进入 MVP。后续扩展 iOS、Android 时，默认按独立本地工作区设计，不承诺与 macOS 自动同步。

后续移动端布局：

- 底部导航。
- 任务卡片。
- 记录卡片。
- 字段折叠。
- 详情页分段 Tabs。
- 关键操作放底部固定操作区。

后续移动端不默认展示复杂横向表格。需要表格时提供：

- 简化字段。
- 横向滚动兜底。
- 导出到 Excel。
- macOS 端继续处理提示。

## 6. 核心交互流程

### 6.1 首次使用

```text
创建本地工作区
  -> 配置 TikHub API Key
  -> 配置至少一个模型供应商
  -> 运行连接测试
  -> 创建第一个采集任务
```

前端要求：

- 每一步都要显示当前状态。
- 密钥输入默认隐藏。
- 连接测试失败要展示供应商、状态码或网络错误摘要。
- 不要求一次配置所有模型供应商。

### 6.2 表单式采集

控件：

- 平台选择。
- 数据类型选择。
- 国家 / 地区类型选择。
- 关键词或账号输入。
- 时间范围。
- 最大记录数。
- 成本上限。
- 预览数量。

前端要求：

- 参数变化后重新估算成本。
- 不支持国家 / 地区筛选时禁用并提示原因。
- 用户确认前不进入正式采集。
- 展示最终传给后端的结构化参数摘要。

### 6.3 自然语言采集

交互：

```text
自然语言输入
  -> AI 解析
  -> 采集计划卡片
  -> 用户编辑
  -> 成本确认
  -> 运行
```

前端要求：

- AI 解析结果必须回填到可编辑表单。
- 不确定字段以待确认状态展示。
- 缺失必要条件时展示追问，不直接运行。
- 用户可以重新生成采集计划。
- 用户可以查看原始自然语言需求和解析版本。

### 6.4 提示词优化

MVP 前端必须提供基础入口，不只是工程内部文件。

页面能力：

- 查看内置提示词模板。
- 查看提示词版本。
- 查看变更说明。
- 查看绑定的输出 Schema。
- 运行基础回归样例。
- 查看回归样例通过、失败和差异摘要。
- 回滚到上一版本。

MVP 不要求：

- 在线 A/B 测试。
- 自动 prompt search。
- 面向普通用户暴露复杂系统提示词编辑器。

推荐权限：

- 默认用户可查看模板版本和回归结果。
- 高级设置中允许编辑自定义模板。
- 编辑后必须运行基础回归样例，否则标记为未验证。

### 6.5 数据查看与追溯

桌面端建议：

- 左侧数据表格。
- 中间记录详情。
- 右侧 AI 结论和证据。

后续移动端建议：

- 记录列表。
- 详情页 Tabs：原始、标准化、AI、证据、日志。

证据视图必须展示：

- 原始记录 ID。
- 原始链接。
- 原始字段摘要。
- AI 输出字段。
- 转换理由。
- 置信度。
- 提示词版本。
- 模型运行记录。

### 6.6 导出

前端导出流程：

```text
选择报告类型
  -> 检查数据完整性
  -> 预览报告
  -> 选择 Excel / PDF
  -> 生成导出任务
  -> 展示进度和结果路径
```

导出前必须提示：

- 部分成功范围。
- 待人工确认字段。
- 失败步骤。
- 是否包含原始响应摘要。
- 是否启用 Webhook。

## 7. 状态管理

### 7.1 TanStack Query

用于：

- 工作区读取。
- 任务列表。
- 任务详情。
- 连接测试。
- 采集计划生成状态。
- 任务运行状态轮询。
- 导出任务状态。
- Webhook 测试。

要求：

- 查询 key 必须包含工作区 ID 和任务 ID。
- 长任务使用轮询或事件订阅。
- 失败状态保留错误详情，不用通用“出错了”。

### 7.2 Zustand

用于：

- 当前主题。
- 当前导航。
- 表格列显示隐藏。
- 右侧抽屉打开状态。
- 当前选中记录。
- 本地草稿。
- 后续移动端底部 sheet 状态。

不用于：

- 持久任务数据。
- 原始记录数据。
- 密钥。
- AI 运行记录。

### 7.3 表单状态

表单必须做到：

- Schema 驱动。
- 字段级错误。
- 提交前预校验。
- 和后端 Schema 一致。
- 自然语言解析结果可回填。

## 8. 前后端通信

前端通过 Tauri invoke 调用本地后端命令。

命令封装建议：

```text
commands/
  workspace.ts
  secrets.ts
  providers.ts
  tikhub.ts
  tasks.ts
  ai.ts
  prompts.ts
  reports.ts
  exports.ts
  webhooks.ts
```

前端要求：

- 不直接调用 TikHub。
- 不直接调用模型供应商。
- 不直接读取系统安全存储。
- 所有敏感操作走 Tauri command。
- 命令返回统一错误结构。

统一错误结构建议：

```ts
type AppError = {
  code: string
  message: string
  stage?: string
  retryable: boolean
  details?: Record<string, unknown>
}
```

## 10. 可访问性与可用性

- 所有输入框必须有 label。
- 键盘可操作。
- Tooltip 不承载关键唯一信息，关键错误必须可见。
- 颜色不能作为唯一状态表达。
- 后续移动端适配安全区。
- 长文本、长链接可换行或截断。
- 表格列宽稳定。
- 加载状态和空状态要明确。

## 11. 前端性能

风险点：

- 大表格渲染。
- 长文本和评论数据。
- 图表重绘。
- PDF 预览。
- 任务状态轮询。

优化策略：

- 表格虚拟滚动。
- 分页加载。
- 详情按需加载。
- 图表数据聚合后渲染。
- 导出任务交给后端执行。
- 长任务减少高频轮询。

## 13. 前端验收标准

- 用户能在 3 分钟内完成 TikHub 和一个模型供应商配置。
- 用户能通过表单创建任务并看到成本估算。
- 用户能通过自然语言生成可编辑采集计划。
- 缺失必要条件时界面展示追问，不进入运行。
- 用户能看到原始数据、标准化数据、AI 结果和证据。
- 用户能运行提示词基础回归样例并看到结果。
- 用户能导出 Excel 和 PDF。
- macOS 窄窗口不会出现关键内容重叠或按钮文字溢出。
