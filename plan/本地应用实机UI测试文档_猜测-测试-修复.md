# 本地应用实机 UI 测试文档：猜测-测试-修复

更新时间：2026-07-06 19:50:15 CST

## 1. 文档目的

本文档用于指导智能数据整理平台 macOS 本地应用的实机 UI 测试。测试对象必须是打包后的桌面应用产物，而不是浏览器开发服务器页面。

核心方法采用“猜测-测试-修复”闭环：

- 猜测：先明确可能出问题的位置、原因和影响范围。
- 测试：用可重复步骤验证猜测是否成立。
- 修复：只修确认存在的问题，修复前先界定改动边界。
- 回归：修复后重新执行原失败用例和关联用例，确认没有引入新问题。

## 2. 测试对象

应用名称：智能数据整理平台

应用类型：Tauri v2 macOS 本地桌面应用

前端目录：`apps/macos`

打包命令：

```bash
pnpm tauri build --bundles app
```

打包产物：

```text
apps/macos/src-tauri/target/release/bundle/macos/智能数据整理平台.app
```

判断标准：

- 测试窗口必须来自 `.app` 打包产物。
- 实机检测中页面地址应为 `tauri://localhost`。
- 不得用 `http://127.0.0.1:5173` 或 `http://localhost:5173` 的浏览器页面代替本地应用测试。

## 3. 测试前准备

1. 进入前端应用目录：

```bash
cd "apps/macos"
```

2. 执行静态验证：

```bash
pnpm lint
pnpm build
```

3. 打包本地应用：

```bash
pnpm tauri build --bundles app
```

4. 打开打包产物：

```bash
open "src-tauri/target/release/bundle/macos/智能数据整理平台.app"
```

5. 确认测试对象：

- 应用窗口标题为“智能数据整理平台”。
- 应用内容来自 `tauri://localhost`。
- 不依赖正在运行的 Vite dev server。

## 4. 猜测-测试-修复流程

### 4.1 猜测

每一个问题都先写成可验证猜测，不直接进入修改。

猜测模板：

```text
猜测 ID：
现象：
可能原因：
影响范围：
验证方式：
预期结果：
优先级：
```

示例：

```text
猜测 ID：H-001
现象：点击数据表格行后，右侧证据面板可能没有同步更新。
可能原因：表格行点击事件未正确更新 selectedRecordId，或右侧面板读取状态滞后。
影响范围：数据核验、来源追溯、人工确认流程。
验证方式：点击 rec-103 行，观察右侧“来源追溯”标题是否从 rec-104 更新为 rec-103。
预期结果：右侧标题、状态、洞察、证据和原始链接均切换到 rec-103。
优先级：高
```

### 4.2 测试

测试步骤必须可复现，避免只写“看起来正常”。

测试模板：

```text
测试 ID：
关联猜测：
前置条件：
操作步骤：
预期结果：
实际结果：
结论：
证据：
```

结论只允许使用以下口径：

- 通过：实际结果与预期一致。
- 失败：实际结果与预期不一致，需要修复。
- 阻塞：无法执行测试，需要先处理环境或依赖问题。
- 不适用：当前版本不包含该能力。

### 4.3 修复

只有当测试结论为“失败”时才进入修复。

修复前必须先记录：

```text
修复 ID：
关联失败测试：
根因判断：
修改范围：
涉及文件：
风险：
需要用户确认：
```

修复原则：

- 先定位根因，再写代码。
- Bug 修复应先补能复现问题的测试，再修到测试通过。
- 如果改动超过 3 个文件，先拆成更小任务。
- 修复范围只覆盖当前失败点，不顺手重构无关模块。
- 修复后必须列出可能破坏的能力和建议补充的测试。

### 4.4 回归

修复后至少执行：

- 原失败用例。
- 同一模块的相邻用例。
- 一条打包应用启动验证。
- 一条无副作用交互验证。

回归模板：

```text
回归 ID：
关联修复：
回归范围：
执行命令：
实机操作：
结果：
残余风险：
```

## 5. 当前版本核心测试清单

### 5.1 打包与启动

猜测：打包产物可能无法启动，或仍依赖开发服务器。

测试步骤：

1. 执行 `pnpm tauri build --bundles app`。
2. 打开 `src-tauri/target/release/bundle/macos/智能数据整理平台.app`。
3. 检查窗口标题与页面地址。

预期结果：

- 打包成功。
- `.app` 可正常启动。
- 页面地址为 `tauri://localhost`。
- 关闭 Vite dev server 后，应用仍可显示界面。

### 5.2 首屏布局

猜测：打包后的 WebView 渲染可能与 Chrome dev server 页面存在差异，导致布局错位或文字溢出。

测试步骤：

1. 打开 `.app`。
2. 检查侧边栏、顶部操作区、指标卡、连接状态、采集创建、右侧详情面板。
3. 观察中文、英文、数字、状态标签是否互相遮挡。

预期结果：

- 首屏核心模块完整可见。
- 没有明显重叠、截断、错位。
- 主要按钮可点击区域清晰。

### 5.3 采集入口 Tab 切换

猜测：Radix Tabs 在 Tauri WebView 中可能出现选中态或内容切换异常。

测试步骤：

1. 点击“自然语言”Tab。
2. 观察自然语言输入框是否出现。
3. 点击“表单式”Tab。
4. 观察平台、数据类型、国家/地区、关键词、时间范围、最大记录数、成本上限字段是否恢复显示。

预期结果：

- Tab 选中态正确切换。
- 内容区域跟随 Tab 变化。
- 切换后采集计划预览不丢失。

### 5.4 表单生成计划

猜测：表单提交可能无法更新采集计划预览，或数值字段转换异常。

测试步骤：

1. 在“表单式”Tab 中保持默认值。
2. 点击“生成计划”。
3. 修改“成本上限”为大于 120 的数值。
4. 再次点击“生成计划”。

预期结果：

- 默认值提交后，采集计划维持有效状态。
- 成本计算正常显示。
- 当成本上限大于 120 时，状态应进入“待人工确认”。

### 5.5 表格与证据面板联动

猜测：选择表格行后，右侧来源追溯面板可能不会同步更新。

测试步骤：

1. 打开应用后确认右侧默认显示 `rec-104`。
2. 点击数据表格中的 `rec-103` 行。
3. 检查右侧“来源追溯”标题、状态、洞察、证据、原始链接。

预期结果：

- 右侧标题更新为 `rec-103`。
- 状态更新为“待人工确认”。
- 洞察更新为语音误触和方言识别相关内容。
- 原始链接更新为 `https://example.local/douyin/103`。

### 5.6 侧边栏导航高亮

猜测：侧边栏按钮只改变选中态，没有实际视图切换；或高亮状态与用户点击不一致。

测试步骤：

1. 点击侧边栏“数据”。
2. 点击侧边栏“任务”。
3. 点击侧边栏“概览”。

预期结果：

- 被点击项高亮。
- 未点击项取消高亮。
- 当前版本如果尚未做分屏路由，也应至少保持高亮状态一致。

### 5.7 导出区无副作用检查

猜测：“执行导出检查”按钮可能被误认为正式导出，或未来接入后可能触发外部传输。

测试步骤：

1. 点击“执行导出检查”。
2. 观察是否出现本地检查状态变化、弹窗或错误。
3. 检查是否有明显外部上传、Webhook 发送、文件覆盖行为。

预期结果：

- 当前版本不应上传敏感数据。
- 若尚未接入导出逻辑，应无破坏性副作用。
- 后续接入真实导出时，应补文件生成路径、覆盖策略和外部传输确认。

## 9. 发布前最低通过标准

发布前至少满足：

- `pnpm lint` 通过。
- `pnpm build` 通过。
- `pnpm tauri build --bundles app` 通过。
- 打包后的 `.app` 能在本机正常启动。
- 首屏无明显布局阻塞问题。
- 采集入口 Tab 可切换。
- 表格选择与右侧证据面板联动正常。
- 不发生未确认的外部传输、文件覆盖、账号权限变更或费用动作。

## 10. 后续建议补充的自动化测试

- 启动冒烟测试：验证 `.app` 可启动并出现主窗口。
- 交互回归测试：Tab 切换、表格选中、表单提交、侧边栏高亮。
- 视觉回归测试：桌面默认窗口、最小窗口、全屏窗口截图对比。
- 导出安全测试：确认导出前检查不会上传密钥、Header、完整原始数据。
- 数据一致性测试：记录表、证据面板、导出内容使用同一条记录 ID 作为关联键。
