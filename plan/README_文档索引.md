## 3. 当前 MVP 决策总表

| 主题 | 决策 |
| --- | --- |
| MVP 端 | 先做 macOS 桌面端 |
| 工作区 | 单用户本地工作区 |
| 数据保存 | 工作区、任务、原始数据、AI 运行记录、报告、导出文件全部保存在本机本地 |
| 账号 | MVP 不做账号体系、登录和注册 |
| 远端数据库 | MVP 不接远端数据库 |
| 远端同步 | MVP 不做远端同步或多设备自动同步 |
| 云端任务 | MVP 不做云端任务、云端长期任务执行或云端队列 |
| API 配置与密钥 | 当前工作区 `secrets/api-config.json` 是 TikHub 与 AI 配置唯一主数据源；密钥以明文保存在其中，目录 `0700`、文件 `0600`，SQLite 仅保存无明文的派生镜像 |
| 旧 Keychain 迁移 | 不读取、不自动删除旧条目；旧 SQLite 配置导入为 `needs_rebind`，当前指针置空，用户需重新输入一次密钥 |
| 数据采集 | TikHub REST API 是 MVP 核心路径 |
| MCP | TikHub MCP 放到 V1 高级 Agent 模式 |
| 平台 | TikTok、抖音、小红书 |
| 后续四端 | Windows、macOS、iOS、Android 各自独立本地工作区运行，不默认跨端同步 |
| 后续账号与远端能力 | 账号功能、远端数据库、云端同步、云端任务、云端密钥托管作为独立版本重新评审 |
| 采集入口 | 表单式精确采集、自然语言智能采集 |
| 自然语言执行 | 当前由本地规则引擎生成计划，应用校验，用户确认后调用 TikHub REST API；已保存 AI 配置不参与执行 |
| AI API 配置 | 支持 OpenAI、Anthropic、Gemini、自定义 OpenAI-compatible 和 Ollama 的多条配置保存、本地完整性校验与当前配置切换；当前不发真实模型请求 |
| TikHub 任务绑定 | 新鲜任务在执行器首次领取时绑定当时的当前 TikHub 配置；运行中与恢复任务继续使用原不可变快照 |
| 提示词优化 | MVP 必做，包含版本管理、Schema、证据引用、基础回归样例 |
| 导出 | Excel 工作簿和 PDF 报告 |
| n8n | 不做核心卖点，通过 Webhook 和文档保留接入 |
| 反数据 slop | 核心洞察必须有来源、证据、运行记录和状态边界 |

## 4. 维护规则

每个决策只保留一个事实源：产品范围和验收归 PRD，跨层架构归《技术框架总览》，前端结构归《前端技术方案》，后端和存储细节归各自专项文档。其他文档只引用，不复制整段实现说明。代码和打包产物用于证明当前实现状态，规划文档不能把目标态写成已完成能力。

### 4.1 修改 PRD 时

如果修改以下产品边界，需要检查技术文档是否仍然成立，但不要把实现细节复制回 PRD：

- MVP 范围。
- 支持平台。
- 自然语言采集流程。
- 模型供应商范围。
- 提示词优化要求。
- 导出格式。
- 本地存储和密钥策略。
- API 配置主数据源、文件权限、旧 Keychain 迁移或任务领取绑定规则。

### 4.2 修改技术方案时

如果技术方案改变产品边界，必须回写 PRD；如果只改变库、目录或实现方式，只更新对应技术文档。

例如：

- 从本地优先改为云端代理。
- 增加账号、远端数据库、远端同步、云端任务或云端密钥托管。
- 让 AI 直接调用 TikHub。
- 把 CSV 改为主导出格式。
- 把 TikHub MCP 提前到 MVP 核心路径。
- 移除提示词优化或证据引用。

### 4.3 修改数据库方案时

必须同步检查：

- 后端技术方案。
- AI 运行记录。
- 导出完整性检查。
- 备份策略。
- 安全测试。

### 4.4 修改前端方案时

必须同步检查：

- 自然语言采集确认流程。
- 提示词回归测试入口。
- 导出和错误展示。
- 打包后 macOS 应用在最小窗口下的实际表现。

## 5. 当前实现基线与剩余缺口

当前实现状态以 `apps/macos` 代码、自动化测试和打包应用为准：

| 能力 | 状态 | 当前依据 |
| --- | --- | --- |
| Tauri v2 macOS 应用与 SQLite 工作区 | 已实现 | `apps/macos/src-tauri` |
| 私有 JSON API 配置注册表 | 已实现 | `api_profiles.rs`、`api_profiles/storage.rs`；JSON 是唯一主数据源，SQLite 无明文且仅作派生镜像 |
| 旧配置无 Keychain 导入 | 已实现 | 旧元数据导入为 `needs_rebind`，不读取或删除 Keychain 条目 |
| TikHub 首次领取绑定与恢复快照 | 已实现 | 新鲜任务领取时绑定当前配置，运行中与恢复任务复用不可变快照 |
| TikHub 三平台能力映射与表单计划 | 已实现 | `collection.rs`、`tikhub.rs` |
| 提示词版本与基础回归记录 | 部分实现 | 已有存储和界面，真实执行门禁仍需加强 |
| Excel 导出 | 已实现 | Rust 后端生成基础多工作表文件 |
| PDF 导出 | 部分实现 | 当前只有最小文件骨架，中文字体、完整报告内容、分页和完整性验收尚未完成 |
| 自然语言计划 | 部分实现 | 当前为本地规则解析，真实模型网关尚未接入 |
| AI API 配置 | 已实现 | 支持多配置保存、本地完整性校验和切换；不执行真实模型调用 |
| 前端信息架构 | 已实现 | 当前收敛为工作台、指南、设置三个一级入口 |

## 6. 测试优先级

最高优先级：

- 密钥不泄露。
- 私有 JSON 权限必须保持目录 `0700`、文件 `0600`，符号链接、宽松权限和损坏 JSON 必须失败关闭。
- 旧配置迁移和运行时不得访问 Keychain，不出现系统钥匙串权限窗。
- 密钥哨兵只能出现在 JSON，不得进入 SQLite、日志、审计、错误、导出、Webhook 或任务快照。
- 新鲜任务首次领取绑定当前 TikHub，恢复任务不得跟随配置切换。
- 自然语言采集不会绕过用户确认。
- AI 不能直接调用 TikHub。
- 提示词输出必须通过 Schema。
- 核心洞察必须能追溯到原始记录。
- Excel 和 PDF 不包含 API Key。

第二优先级：

- TikTok、抖音、小红书三平台采集可用。
- 自定义 OpenAI-compatible endpoint 可用。
- Webhook 成功和失败有日志。
- macOS 桌面端无布局重叠。
- 本地备份可恢复。
