## 3. 当前 MVP 决策总表

| 主题 | 决策 |
| --- | --- |
| MVP 端 | 先做 macOS 桌面端 |
| 工作区 | 单用户本地工作区 |
| 数据保存 | 工作区、任务、原始数据、AI 运行记录、报告、导出文件全部保存在本机本地 |
| 账号 | MVP 不做账号体系、登录和注册 |
| 远端数据库 | MVP 不接远端数据库 |
| 远端同步 | MVP 不做远端同步或多设备自动同步 |
| 云端任务 | MVP 不做云端任务、云端长期任务执行或云端队列 |
| 密钥 | 仅本地安全存储持久化，不做云端密钥托管，不上传自有服务器或远端数据库 |
| 数据采集 | TikHub REST API 是 MVP 核心路径 |
| MCP | TikHub MCP 放到 V1 高级 Agent 模式 |
| 平台 | TikTok、抖音、小红书 |
| 后续四端 | Windows、macOS、iOS、Android 各自独立本地工作区运行，不默认跨端同步 |
| 后续账号与远端能力 | 账号功能、远端数据库、云端同步、云端任务、云端密钥托管作为独立版本重新评审 |
| 采集入口 | 表单式精确采集、自然语言智能采集 |
| 自然语言执行 | AI 生成计划，应用校验，用户确认，应用调用 TikHub REST API |
| 模型供应商 | 内置 OpenAI、Gemini、Claude，支持扩展供应商和自定义 OpenAI-compatible |
| 提示词优化 | MVP 必做，包含版本管理、Schema、证据引用、基础回归样例 |
| 导出 | Excel 工作簿和 PDF 报告 |
| n8n | 不做核心卖点，通过 Webhook 和文档保留接入 |
| 反数据 slop | 核心洞察必须有来源、证据、运行记录和状态边界 |

## 4. 维护规则

### 4.1 修改 PRD 时

如果修改以下内容，需要同步检查技术文档：

- MVP 范围。
- 支持平台。
- 自然语言采集流程。
- 模型供应商范围。
- 提示词优化要求。
- 导出格式。
- 本地存储和密钥策略。

### 4.2 修改技术方案时

如果技术方案改变产品边界，必须回写 PRD。

例如：

- 从本地优先改为云端代理。
- 增加账号、远端数据库、远端同步、云端任务或云端密钥托管。
- 让 AI 直接调用 TikHub。
- 把 CSV 改为主导出格式。
- 把 TikHub MCP 提前到 MVP 核心路径。
- 移除提示词优化或证据引用。

### 4.3 修改数据库方案时

必须同步检查：

- 后端技术方案。
- AI 运行记录。
- 导出完整性检查。
- 备份策略。
- 安全测试。

### 4.4 修改前端方案时

必须同步检查：

- PRD 的前端设计指南。
- 自然语言采集确认流程。
- 提示词回归测试入口。
- 移动端适配。
- 导出和错误展示。

## 5. 实现前建议

进入开发前建议先完成：

1. 确认 Tauri v2 macOS 工程模板。
2. 确认 SQLite 访问库和迁移工具。
3. 确认系统安全存储方案。
4. 确认 Provider Registry 初始供应商列表。
5. 确认 TikHub 三个平台的 API 能力映射。
6. 确认提示词模板和基础回归样例。
7. 确认 Excel 和 PDF 的最小报告数据模型。
8. 确认前端页面信息架构。

## 6. 测试优先级

最高优先级：

- 密钥不泄露。
- 自然语言采集不会绕过用户确认。
- AI 不能直接调用 TikHub。
- 提示词输出必须通过 Schema。
- 核心洞察必须能追溯到原始记录。
- Excel 和 PDF 不包含 API Key。

第二优先级：

- TikTok、抖音、小红书三平台采集可用。
- 自定义 OpenAI-compatible endpoint 可用。
- Webhook 成功和失败有日志。
- macOS 桌面端无布局重叠。
- 本地备份可恢复。
